# Everlight Ventures — Style Guide

## Writing Tone
- Professional but approachable — we explain things clearly, not condescendingly
- Confident and knowledgeable without being preachy
- Direct. Get to the point. No fluff paragraphs.
- Use "you" and "we" — never "one should" or passive voice
- Conversational when possible, formal when credibility matters

## Formatting Rules
- **Headings:** Use H1 for title, H2 for major sections, H3 for subsections. Never skip levels.
- **Bullets:** Use dashes (-) for lists, not asterisks. Keep items parallel in structure.
- **Bold:** Use for key terms on first mention and for emphasis. Don't overuse.
- **Numbers:** Spell out one through nine. Use digits for 10+. Always use digits with units ($5, 3%).
- **Links:** Use descriptive anchor text. Never "click here."

## Content Structure
1. Hook — why should the reader care? (1-2 sentences)
2. Context — what they need to know (brief background)
3. Meat — the actual value (steps, comparison, explanation)
4. Action — what to do next (CTA, next steps)

## Vocabulary
- Use: straightforward, simple, clear, guide, build, grow
- Avoid: synergy, leverage (as verb), disrupt, game-changing, revolutionary
- For crypto/trading: use plain terms, always define jargon on first use
- For kids content: age-appropriate vocabulary (ages 4-8), short sentences

## Platform Adaptations
- **Blog:** 1200-1800 words, structured with H2s, scannable
- **Instagram:** Visual-first, 150-200 word captions, 3-5 hashtags
- **TikTok:** Hook in 2 seconds, casual tone, 7-10 hashtags
- **X/Twitter:** Under 280 chars, punchy, thread-ready
- **Email:** Under 300 words, one clear CTA, mobile-friendly
- **Facebook:** 200-300 words, link included, business value focus

## File Naming
- Use lowercase with underscores for code files: `daily_report.md`
- Use lowercase with dashes for slugs: `how-to-pick-an-xlm-wallet`
- Date folders: YYYY/MM/DD format
- Series folders: lowercase with underscores: `adventures_with_sam`
