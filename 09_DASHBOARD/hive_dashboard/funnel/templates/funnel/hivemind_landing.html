Workflow

How the hive actually works.

Every command moves through the same sequence: classify, route, execute, surface risk, then log the result back into the operating system. That is what turns a handful of smart tools into a real machine.

1. Route the job

Hive decides whether the request belongs to coding, content, broker ops, automation, or trading review.

2. Pull live context

MCP resources, RAG memory, Supabase data, and workspace manifests give every agent the same operating picture.

3. Close the loop

n8n, Stripe, and the watchtower publish outcomes back to the board so you are informed without babysitting tabs.
